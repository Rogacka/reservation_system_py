# reservation_system_py

## Технічне завдання для проекту системи бронювання

Проект системи бронювання призначений для розробки веб-додатку на базі
Django, який дозволить користувачам бронювати кімнати або місця для
різних заходів. Система надаватиме інтерфейси для кінцевих користувачів
для бронювання, а також для адміністраторів для керування доступністю та
бронюваннями.

## Основні функціональні вимоги

**1. Вибір дати та часу бронювання**
Можливість вибору дати початку та завершення бронювання.
Перевірка доступності обраного періоду в реальному часі.

**2. Реєстрація бронювань**
Форма бронювання з полями: ім'я користувача, електронна пошта,
вибір кімнати/місця, дати початку та завершення бронювання.
Підтвердження бронювання через електронну пошту.

**3. Керування бронюваннями**
Панель адміністратора для перегляду, підтвердження, відміни
бронювань.
Можливість редагування та видалення бронювань.

**4. Перегляд доступності**
Календар доступності для визначення вільних та зайнятих періодів.
Фільтрація за типами кімнат/місць.

**5. Керування кімнатами/місцями**
Додавання, редагування, видалення інформації про кімнати/місця.


## Технічні вимоги
- **Backend:** Django Framework для розробки серверної частини.
- **Frontend:** HTML, CSS, JavaScript (можливо з використанням фреймворків
типу Bootstrap для стилізації).
- **База даних:** PostgreSQL або SQLite для розробки.
- **Аутентифікація та авторизація:** Використання стандартних засобів Django
для управління користувачами та їхніми правами.
Мінімально 3 моделі
